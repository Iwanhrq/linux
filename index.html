O que é Linux?

O Linux, à semelhança do Windows (Microsoft) e do macOS (Apple), é um sistema operacional baseado em Unix desenvolhido para desktops, mas que também é utilizado em servidores, smartphones, tablets e outras categorias de dispositivos, como caixas bancárias. Diferentemente de seus concorrentes mais renomados, o Linux não teve sua origem no âmbito comercial, sendo que seu software e desenvolvimento são realizados com código aberto, o que implica que qualquer indivíduo pode criar e distribuir aplicativos para ele.

A estrutura fundamental do Linux consiste em um kernel, um software elaborado para possibilitar a comunicação entre outros programas e interpretar comandos para a unidade de processamento e outros componentes eletrônicos. Entretanto, para seu funcionamento adequado, também são necessários aplicativos específicos e as bibliotecas correspondentes a esses aplicativos.

Isso implica que um utilizador de Linux pode optar entre diversas aplicações para realizar a mesma tarefa, seja ela edição de texto, interfaces gráficas ou mesmo prompts de comando. Esse processo é semelhante à escolha entre Chrome e Firefox: ambos são navegadores, capazes de executar as mesmas funções, mas apresentando diferentes formas e aparências.

A origem

O nome Linux surgiu da mistura de Linus + Unix. Linus é o nome do criador do Linux, Linus Torvalds. E Unix, é o nome de um sistema operacional de grande porte, no qual contaremos sua história agora, para que você entenda melhor a do Linux. A origem do Unix tem ligação com o sistema operacional Multics, projetado na década de 1960. Esse projeto era realizado pelo Massachusets Institute of Technology (MIT), pela General Eletric (GE) e pelos laboratórios Bell (Bell Labs) e American Telephone na Telegraph (AT&T). A intenção era de que o Multics tivesse características de tempo compartilhado (vários usuários compartilhando os recursos de um único computador), sendo assim, o sistema mais arrojado da época. Em 1969, já exisita uma versão do Multics rodando num computador GE645.

Ken Thompsom era um pesquisador do Multics e trabalhava na Bell Labs. No entanto, a empresa se retirou do projeto tempos depois, mas ele continuou seus estudos no sistema. Desde então, sua idéia não era continuar no Multics original e sim criar algo menor, mas que conservasse as idéias básicas do sistema. A partir daí, começa a saga do sistema Unix. Brian Kernighan, também pesquisador da Bell Labs, foi quem deu esse nome.

Em 1973, outro pesquisador da Bell Labs, Dennis Ritchie, rescreveu todo o sistema Unix numa linguagem de alto nível, chamada C, desenvolvida por ele mesmo. Por causa disso, o sistema passou a ter grande aceitação por usuários externos à Bell Labs. Entre 1977 e 1981, a AT&T, alterou o Unix, fazendo algumas mudanças particulares e lançou o System III. Em 1983, após mais uma série de modificações, foi lançado o conhecido Unix System IV, que passou a ser vendido. Até hoje esse sistema é usado no mercado, tornando-se o padrão internacional do Unix. Esse sistema é comercializado por empresas como IBM, HP, Sun, etc. O Unix, é um sistema operacional muito caro e é usado em computadores poderosos (como mainframes) por diversas multinacionais.

Os softwares de Windows são todos compatíveis com o Linux?

Não, os softwares desenvolvidos para Windows não são automaticamente compatíveis com o Linux. Isso ocorre devido a diferenças fundamentais entre os dois sistemas operacionais, como a arquitetura do sistema, as bibliotecas e os padrões de API (Interface de Programação de Aplicativos).

Em resumo, a compatibilidade entre softwares do Windows e Linux varia e geralmente requer esforço adicional para tornar os aplicativos do Windows executáveis no Linux. É recomendável verificar se há alternativas nativas para Linux ou considerar o uso de emuladores ou máquinas virtuais quando necessário.

O que é root?

A palavra inglesa root significa raiz. E no mundo tecnológico, o termo root é usado para caracterizar o processo que permite o uso de um sistema operacional baseado no Unix/Linux como superadministrador.

E como o sistema operacional Android é baseado no Linux, dá para fazer root e administrar todas as funções do smartphone, com acesso à sua "raiz", o que pode trazer diversos benefícios. Por segurança, os celulares Android vêm com uma proteção, que limita o acesso do usuário comum a alguns dados, como pastas importantes do sistema, por exemplo. Assim, você não corre o risco de excluir ou modificar algo acidentalmente, podendo até fazer com que o smartphone não funcione mais. Mas, fazendo root, o usuário passa a ter controle total de todas as funções do sistema.

O que são respositórios?

Em um contexto de sistemas Linux, um "repositório" refere-se a um local centralizado onde o software é armazenado e gerenciado. Isso permite que os usuários instalem, atualizem e removam programas e pacotes de software de forma organizada e eficiente. Os repositórios são uma parte fundamental dos sistemas de gerenciamento de pacotes usados em distribuições Linux, como o APT (Advanced Package Tool) usado no Debian e no Ubuntu, o YUM (Yellowdog Updater, Modified) usado no Red Hat e no CentOS, e o Zypper usado no openSUSE, entre outros.

Em resumo, os repositórios são uma parte fundamental do ecossistema de software do Linux, permitindo que os usuários instalem, atualizem e gerenciem o software de maneira conveniente e segura.

O que são KDE e Gnome? Qual é melhor?

O KDE e o GNOME são dois dos ambientes de desktop mais populares para sistemas operacionais baseados em Linux e Unix. Eles fornecem interfaces gráficas de usuário (GUI) para interagir com o sistema operacional e os aplicativos. Ambos têm suas próprias abordagens de design e funcionalidade, e a escolha entre eles geralmente se resume a preferências pessoais.

Quanto qual é melhor, não existe uma resposta única. A escolha entre o KDE e o GNOME é altamente subjetiva e depende das preferências do usuário. Alguns preferem o KDE pela sua flexibilidade e pela quantidade de opções de personalização, enquanto outros preferem o GNOME pela sua simplicidade e design elegante.

A melhor maneira de determinar qual ambiente de desktop é ideal para você é experimentar ambos e ver qual deles se adapta melhor às suas necessidades e estilo de uso. A maioria das distribuições Linux permite que você escolha entre o KDE e o GNOME durante a instalação ou após a instalação, para que você possa experimentar ambos antes de tomar uma decisão.

Passos para a Instalação do Linux

Preciso apagar meu windows para instalar o linux?

Não é necessário apagar o Windows para instalar o Linux no mesmo computador. Você pode ter ambos os sistemas operacionais coexistindo no mesmo sistema. Isso é conhecido como "dual boot". O dual boot permite que você escolha qual sistema operacional deseja usar sempre que iniciar o computador.

Lembre-se de que a configuração de um dual boot requer algum conhecimento e cuidado, especialmente ao mexer nas partições do disco rígido. Certifique-se de seguir as instruções da distribuição Linux escolhida e faça backup de seus dados antes de começar. Além disso, esteja ciente de que as ações que você toma durante o processo de instalação do Linux podem afetar tanto o Windows quanto o Linux, por isso, é importante seguir as instruções com atenção.

Posso testar o Linux sem instalar?

Sim, você pode testar o Linux sem a necessidade de instalá-lo no seu computador. Isso é possível graças a distribuições Linux que são projetadas para serem "Live" ou "Live CD/USB". Essas distribuições são criadas de forma a permitir que você execute o Linux a partir de um CD, DVD ou unidade USB sem a necessidade de instalação.

Como instalar programas no linux?

Para instalar programas no Linux, você usa uma ferramenta chamada "gerenciador de pacotes". É como uma loja de aplicativos. Você digita um comando para buscar e instalar programas. Cada distribuição Linux pode usar comandos diferentes, mas o processo geral é:
- Abra o Terminal, que é como uma janela de comando.
- Digite o comando para atualizar a lista de programas disponíveis. Isso ajuda a ter a lista mais recente.
- Em seguida, use outro comando para instalar o programa desejado. Isso fará o download e a instalação automaticamente.
- Espere até que a instalação esteja concluída.
- Agora, o programa está instalado e pronto para ser usado. Você pode digitá-lo no Terminal ou procurá-lo no menu de aplicativos.

O que é uma distribuição Linux?

Uma distribuição Linux® é um sistema operacional instalável baseado no kernel do Linux que dá suporte a bibliotecas, repositórios e programas do usuário. Cada versão da comunidade e do fornecedor é considerada uma distribuição. O sistema operacional Linux é open source e disponibilizado sob a Licença Pública Geral GNU (GPL). Por isso, qualquer pessoa pode executar, analisar, modificar e redistribuir o código-fonte ou até mesmo vender cópias dessas alterações. Em comparação, os sistemas tradicionais como Unix, Microsoft Windows e MacOS são proprietários e permitem muito menos modificações.

Como escolher uma distribuição Linux?

Escolher a distribuição Linux certa depende do seu caso de uso e dos requisitos de ferramenta. Determinadas distribuições Linux funcionam melhor para diferentes finalidades. Algumas distribuições sãoprojetadas para ambientes de desktop Linux (como XFCE, Raspberry Pi e Cinnamon), enquanto outras são projetadas para dar suporte a sistemas de TI de back-end (como servidores web ou empresariais). Ao escolher sua distribuição Linux, analise primeiro se você precisa de uma empresarial ou da comunidade.

Distribuição Kali Linux

O Kali Linux é uma distribuição de sistema operacional baseada no Debian, projetada especificamente para fins de segurança cibernética e testes de penetração. Ele é mantido e desenvolvido pela Offensive Security, uma empresa especializada em segurança da informação. O Kali Linux é uma das distribuições Linux mais populares para profissionais de segurança cibernética e hackers éticos, pois fornece uma ampla gama de ferramentas e recursos projetados para testar a segurança de sistemas e redes.

É importante observar que o Kali Linux é uma ferramenta poderosa, mas deve ser usada de forma ética e legal. O uso inadequado ou ilegal do Kali Linux pode resultar em consequências legais graves. É destinado a profissionais de segurança cibernética, pesquisadores de segurança e hackers éticos que têm permissão para testar sistemas e redes para identificar vulnerabilidades e fortalecer a segurança.

Linux é melhor que Windows?

A escolha entre o Linux e o Windows depende das suas necessidades específicas. Muitas pessoas usam ambos os sistemas operacionais em diferentes contextos, aproveitando as vantagens de cada um. Se você estiver considerando mudar para o Linux, é aconselhável testá-lo primeiro em um ambiente de dual boot ou em uma máquina virtual para ver se atende às suas necessidades.

Tanto o Windows, quanto o Linux possuem desvantagens e vantagens. Uma desvantagem do Windows são as atualizações forçadas, e uma vantagem é a compatibilidade de software, isto é, a maioria dos programas são desenvolvidos para o Windows. Em Linux, uma vantagem é a segurança, pois tem menor vulnerabilidade a malware, agora uma desvantagem é a compatibilidade de software, pois a maioria dos programas não são desenvolvidos para o Linux, e sim para o Windows.

O Linux não pega vírus?

Não é estritamente verdade que o Linux seja imune a vírus, mas é mais preciso dizer que o Linux é menos suscetível a vírus e malware em comparação com sistemas operacionais como o Windows. Por exemplo, o Linux tem menor participação no mercado, então ele não é muito visado. Ele possui um modelo de segurança mais robusto, isso é muito bom para quem trabalhar com cyberseguranaça e também PARA hackers-éticos, mas isso também pode abrir portas para hackers mal-intecionados, uma vez que a maioria dos hackers mal-intecionados utilizam o Linux.

No entanto, não é correto afirmar que o Linux é completamente imune a vírus e malware. À medida que o Linux se torna mais popular em ambientes corporativos e em servidores, a atenção dos criadores de malware pode aumentar. Além disso, os ataques direcionados a sistemas Linux podem ocorrer, especialmente em servidores. Portanto, é importante manter seu sistema Linux atualizado, usar boas práticas de segurança e ter um software antivírus adequado, especialmente em ambientes de servidor.
